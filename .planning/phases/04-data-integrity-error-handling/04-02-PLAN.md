1. After the dedup check (around line 106), declare `let shouldDecrementOnError = false;`

2. Set `shouldDecrementOnError = true` immediately after the message record is created (after line 118 `executeTakeFirstOrThrow()`). This is the point of no return — once a message record exists, we MUST decrement the counter whether the send succeeds or fails.

3. Wrap the remaining logic (from merge tag processing through completion check) in a `try { ... } finally { ... }` block.

4. In the success path, after the existing Lua decrement (line 193-197) and completion check (lines 199-216), set `shouldDecrementOnError = false` — the counter was already decremented.

5. In the `finally` block:
```typescript
finally {
  if (shouldDecrementOnError) {
    try {
      const shouldComplete = await redis.eval(
        DECR_AND_CHECK_LUA, 1, `twmail:remaining:${campaignId}`
      ) as number;
      if (shouldComplete === 1) {
        await db
          .updateTable('campaigns')
          .set({ status: CampaignStatus.SENT, send_completed_at: new Date() })
          .where('id', '=', campaignId)
          .where('status', '=', CampaignStatus.SENDING)
          .execute();
      }
    } catch (finallyErr) {
      console.error('Failed to decrement counter in finally block', {
        err: finallyErr,
        campaignId,
        contactId,
      });
    }
  }
}
```

6. Also update `worker.on('failed')` (line 230-232) to include job data context:
```typescript
worker.on('failed', (job, err) => {
  const data = job?.data;
  console.error('Bulk send job failed', {
    jobId: job?.id,
    campaignId: data?.campaignId,
    contactId: data?.contactId,
    error: err.message,
  });
});
```

CRITICAL ANTI-PATTERNS TO AVOID:
- Do NOT decrement inside `finally` unconditionally — success path already decrements. Use the `shouldDecrementOnError` flag.
- Do NOT add `total_failed` column — schema does not have it, and DATA-07 does not require it. The key metric is: `total_sent` only counts confirmed SES sends, and the campaign completes correctly.
- Do NOT move the completion check (resend trigger) into the `finally` block's success path — only the basic status transition is needed there. The resend trigger is optional and acceptable to skip on error-path completion.
- Do NOT swallow errors inside the `finally` try/catch — always log with context.
